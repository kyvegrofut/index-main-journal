# index-main-journal

这是一个用于归档和发布多个独立 HTML 页面的仓库。它不针对某个特定域名或网站，而是作为一个通用的页面集合空间存在。

## 项目简介

本仓库主要存放各种独立的 HTML 页面文件。这些页面可以是工具、笔记、演示页、临时页面或其他任何形式的静态内容。所有页面彼此独立，互不依赖，便于直接通过 GitHub Pages 或类似服务访问。

## 目录说明

- `/` — 仓库根目录，通常包含首页 `index.html` 及少量通用文件。
- `/pages/` — 存放主要 HTML 页面文件的目录（可选，可根据需要自行组织）。
- `/assets/` — 存放 CSS、JavaScript、图片等静态资源文件（可选）。
- `/docs/` — 存放与页面相关的说明文档或附加资料（可选）。

> 目录结构并非固定，可根据实际需求进行调整。每个 HTML 页面都应尽量自包含，减少对外部资源的依赖。

## 页面归档说明

仓库中的每个 HTML 页面均可独立访问。建议遵循以下原则：

- 每个页面文件命名清晰，如 `tool.html`、`note-2024.html`。
- 页面内尽量不使用绝对路径引用外部资源，优先使用相对路径或内联资源。
- 页面标题、描述等元信息应简洁明了，便于识别。

## 维护说明

- 新增页面：直接将 HTML 文件放入仓库对应目录，并更新 `index.html` 或其他导航文件（如有）。
- 修改页面：直接编辑对应 HTML 文件，提交并推送至远程仓库。
- 删除页面：移除相关文件，并清理所有引用链接。
- 建议每次提交附带简要的提交信息，说明新增、修改或删除的内容。
